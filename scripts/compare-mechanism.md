# 截图对比机制（已固化）

适用范围：竞品分析截图（分期乐/奇富借条/安逸花/小赢/度小满）

## 1) 时间窗口选择
- 每个 app、每个维度目录（APP/风控/客服/消金/运营）内：
- 读取所有纯数字日期文件夹，按升序排序，取**最早两个**作为对比窗口。

## 2) 同位点匹配规则（关键）
- 文件名先做标准化：
  - 去掉开头日期前缀（如 `0323首页` → `首页`，`0402首页2` → `首页2`）
  - 去空格
- 用标准化后的名字做 1:1 配对。
- 若无法配对：标记“待复核”，禁止直接给高影响。

## 3) 图像对比规则
- 先统一分辨率（取两图最小宽高并 resize）
- 裁掉顶部状态栏区域（默认高度 10%）
- 使用视觉差异均值（像素绝对差均值）计算 diff

## 4) 影响等级映射（仅按截图变化幅度）
- 低：diff < 0.055
- 中：0.055 ≤ diff < 0.12
- 高：diff ≥ 0.12

## 5) 新增/下线特判
- 在新日期出现、老日期无同位点：标记“新增模块”，影响=高
- 老日期存在、新日期缺失：标记“下线/缺失”，不算新增高变化

## 6) 文案约束
- compare 文案必须包含：
  - 对照窗口（如 0323 vs 0402）
  - 忽略状态栏时间
  - 是否同位配对成功
  - 差异幅度（diff）

## 7) 发布前闸门
- 若存在“高影响”但未配对成功：阻断发布
- 若 evidence URL 为 `无` 或非法路径：清理并标缺图
