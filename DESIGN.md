# 设计系统：Stripe

## 1. 视觉主题与氛围

Stripe 的网站是金融科技设计的黄金标准——一个同时兼具技术感与奢华感、精确感与温暖感的系统。页面以干净的白色画布（`#ffffff`）展开，配以深海军蓝标题（`#061b31`）和标志性的紫色（`#533afd`），后者既是品牌锚点，也是交互强调色。这不是企业软件那种冰冷、临床感的紫色，而是一种浓郁、饱和的紫罗兰色，传达出自信与高端感。整体印象就像是一家由世界级字体铸造厂重新设计的金融机构。

定制的 `sohne-var` 可变字体是 Stripe 视觉身份的决定性元素。每个文本元素都启用了 OpenType `"ss01"` 样式集，该样式集修改字符形状，呈现出独特的几何感和现代感。在展示尺寸（48px-56px）下，sohne-var 使用 300 字重——对于标题来说这是极其轻盈的字重，营造出一种空灵、几乎像是在耳语般的权威感。这与"粗体主标题"的惯例恰恰相反；Stripe 的标题感觉不需要大声喧哗。负字间距（56px 时为 -1.4px，48px 时为 -0.96px）将文字收紧成紧密、经过工程化处理的块状。在较小尺寸下，系统同样使用 300 字重，并按比例减少字距，同时通过 `"tnum"` 使用等宽数字来展示金融数据。

真正让 Stripe 与众不同的是其阴影系统。大多数网站采用扁平或单层阴影方案，而 Stripe 使用多层、蓝调阴影：标志性的 `rgba(50,50,93,0.25)` 与 `rgba(0,0,0,0.1)` 结合，创造出带有冷调、几乎带有大气深度感的阴影——就像元素漂浮在暮色天空中。主阴影颜色（50,50,93）的蓝灰色底色与海军蓝-紫色品牌调色板直接相关，即使是层次感也体现出品牌特色。

**核心特征：**
- 所有文本使用 sohne-var 并启用 OpenType `"ss01"`——定义品牌字形的自定义样式集
- 300 字重作为标志性标题字重——轻盈、自信、反常规
- 展示尺寸使用负字间距（56px 时为 -1.4px，向下逐渐放宽）
- 使用 `rgba(50,50,93,0.25)` 的蓝调多层阴影——层次感体现品牌色调
- 使用深海军蓝（`#061b31`）标题而非黑色——温暖、高端、金融级
- 保守的 border-radius（4px-8px）——不使用药丸形状，也不尖锐
- Ruby（`#ea2261`）和 Magenta（`#f96bee`）作为渐变和装饰元素的强调色
- `SourceCodePro` 作为代码和技术标签的等宽字体伴侣

## 2. 色彩调色板与角色

### 主色
- **Stripe Purple**（`#533afd`）：主品牌色，CTA 背景，链接文本，交互高亮。一种饱和的蓝紫罗兰色，作为整个系统的锚点。
- **Deep Navy**（`#061b31`）：`--hds-color-heading-solid`。主标题颜色。不是黑色，不是灰色——一种带有温暖感和深度感的深蓝色。
- **Pure White**（`#ffffff`）：页面背景，卡片表面，深色背景上的按钮文本。

### 品牌色与深色
- **Brand Dark**（`#1c1e54`）：`--hds-color-util-brand-900`。深靛蓝色，用于深色区块、页脚背景和沉浸式品牌时刻。
- **Dark Navy**（`#0d253d`）：`--hds-color-core-neutral-975`。最深的中性色——带有蓝色底调的近黑色，在避免刺眼的同时提供最大深度。

### 强调色
- **Ruby**（`#ea2261`）：`--hds-color-accentColorMode-ruby-icon-solid`。暖红粉色，用于图标、提醒和强调元素。
- **Magenta**（`#f96bee`）：`--hds-color-accentColorMode-magenta-icon-gradientMiddle`。鲜艳的粉紫色，用于渐变和装饰性高亮。
- **Magenta Light**（`#ffd7ef`）：`--hds-color-util-accent-magenta-100`。带有色调的表面，用于 magenta 主题的卡片和徽章。

### 交互色
- **Primary Purple**（`#533afd`）：主链接颜色，激活状态，选中元素。
- **Purple Hover**（`#4434d4`）：用于主元素悬停状态的深紫色。
- **Purple Deep**（`#2e2b8c`）：`--hds-color-button-ui-iconHover`。用于图标悬停状态的深紫色。
- **Purple Light**（`#b9b9f9`）：`--hds-color-action-bg-subduedHover`。柔和淡紫色，用于弱化悬停背景。
- **Purple Mid**（`#665efd`）：`--hds-color-input-selector-text-range`。范围选择器和输入高亮颜色。

### 中性色阶
- **Heading**（`#061b31`）：主标题，导航文本，强标签。
- **Label**（`#273951`）：`--hds-color-input-text-label`。表单标签，次级标题。
- **Body**（`#64748d`）：次级文本，描述，说明文字。
- **Success Green**（`#15be53`）：状态徽章，成功指示器（背景/边框使用 0.2-0.4 透明度）。
- **Success Text**（`#108c3d`）：成功徽章文本颜色。
- **Lemon**（`#9b6829`）：`--hds-color-core-lemon-500`。警告和高亮强调色。

### 表面与边框
- **Border Default**（`#e5edf5`）：卡片、分隔线和容器的标准边框颜色。
- **Border Purple**（`#b9b9f9`）：按钮和输入框的激活/选中状态边框。
- **Border Soft Purple**（`#d6d9fc`）：次级元素的柔和紫调边框。

- **Border Dashed**（`#362baa`）：放置区和占位元素的虚线边框。

### 阴影颜色
- **Shadow Blue**（`rgba(50,50,93,0.25)`）：标志性颜色——蓝调主阴影颜色。
- **Shadow Dark Blue**（`rgba(3,3,39,0.25)`）：用于提升元素的深蓝阴影。
- **Shadow Black**（`rgba(0,0,0,0.1)`）：用于增强深度感的次级阴影层。
- **Shadow Ambient**（`rgba(23,23,23,0.08)`）：用于微妙层次感的柔和环境阴影。
- **Shadow Soft**（`rgba(23,23,23,0.06)`）：用于轻微浮起感的最小环境阴影。

## 3. 排版规则

### 字体族
- **主字体**：`sohne-var`，回退字体：`SF Pro Display`
- **等宽字体**：`SourceCodePro`，回退字体：`SFMono-Regular`
- **OpenType 特性**：所有 sohne-var 文本全局启用 `"ss01"`；金融数据和说明文字的等宽数字使用 `"tnum"`。

### 原则
- **轻盈字重作为标志**：展示尺寸使用 300 字重是 Stripe 最具辨识度的排版选择。当其他人使用 600-700 来吸引注意力时，Stripe 将轻盈感作为奢华——文本如此自信，不需要用字重来确立权威。
- **ss01 无处不在**：`"ss01"` 样式集是不可妥协的。它修改特定字形（可能是替代的 `a`、`g`、`l` 形式），在所有 sohne-var 文本中创造更几何、更现代的感觉。
- **两种 OpenType 模式**：展示/正文文本使用 `"ss01"`，金融数据中的等宽数字使用 `"tnum"`。这两者从不重叠——段落中的数字使用 ss01，数据表格中的数字使用 tnum。
- **渐进式字距**：字间距随尺寸成比例收紧：56px 时为 -1.4px，48px 时为 -0.96px，32px 时为 -0.64px，26px 时为 -0.26px，16px 及以下为正常。
- **双字重简约**：主要使用 300（正文和标题）和 400（UI/按钮）。主字体不使用粗体（700）——SourceCodePro 使用 500/700 来形成代码对比。

## 5. 布局原则

### 间距系统
- 基础单位：8px
- 刻度：1px, 2px, 4px, 6px, 8px, 10px, 11px, 12px, 14px, 16px, 18px, 20px
- 备注：刻度在小端非常密集（从 4px 到 12px 每 2px 一级），反映了 Stripe 面向金融数据的精确 UI 设计

### 网格与容器
- 最大内容宽度：约 1080px
- Hero：居中单列，留有充足内边距，轻盈的标题
- 功能区块：2-3 列网格用于功能卡片
- 使用 `#1c1e54` 背景的全宽深色区块用于品牌沉浸
- 代码/仪表板预览作为带有蓝调阴影的独立卡片

### 留白哲学
- **精确间距**：与极简系统的大片空白不同，Stripe 使用经过测量、有目的性的留白。每个间隙都是经过深思熟虑的排版选择。
- **密集数据，宽松边框**：金融数据显示（表格、图表）紧凑排列，但其周围的 UI 边框却很宽松。这创造了一种受控密度感——就像放在精美框架中组织良好的电子表格。
- **区块节奏**：白色区块与深色品牌区块（`#1c1e54`）交替出现，创造戏剧性的明暗节奏，在不引入任意颜色的情况下避免单调。

### Border Radius 刻度
- Micro（1px）：细粒度元素，微妙圆角
- Standard（4px）：按钮，输入框，徽章，卡片——主力规格
- Comfortable（5px）：标准卡片容器
- Relaxed（6px）：导航，较大的交互元素
- Large（8px）：特色卡片，hero 元素
- 复合：`0px 0px 6px 6px` 用于底部圆角容器（标签面板，下拉菜单底部）

## 6. 深度与层次

| 级别 | 处理方式 | 用途 |
|------|----------|------|
| Flat（Level 0） | 无阴影 | 页面背景，行内文本 |
| Ambient（Level 1） | `rgba(23,23,23,0.06) 0px 3px 6px` | 微妙卡片浮起，悬停提示 |
| Standard（Level 2） | `rgba(23,23,23,0.08) 0px 15px 35px` | 标准卡片，内容面板 |
| Elevated（Level 3） | `rgba(50,50,93,0.25) 0px 30px 45px -30px, rgba(0,0,0,0.1) 0px 18px 36px -18px` | 特色卡片，下拉菜单，弹出框 |
| Deep（Level 4） | `rgba(3,3,39,0.25) 0px 14px 21px -14px, rgba(0,0,0,0.1) 0px 8px 17px -8px` | 模态框，浮动面板 |
| Ring（无障碍） | `2px solid #533afd` 轮廓 | 键盘焦点环 |

**阴影哲学**：Stripe 的阴影系统建立在色彩深度原则之上。大多数设计系统使用中性灰色或黑色阴影，而 Stripe 的主阴影颜色（`rgba(50,50,93,0.25)`）是一种深蓝灰色，呼应品牌的海军蓝调色板。这创造的阴影不仅增加了深度——还增加了品牌氛围。多层方法将这种蓝调阴影与纯黑色次级层（`rgba(0,0,0,0.1)`）以不同的偏移量配对，创造类似视差的深度感，品牌色阴影离元素较远，中性阴影离元素较近。负扩散值（-30px, -18px）确保阴影不会在水平方向延伸超出元素的边界，保持层次感的垂直性和可控性。

### 装饰性深度
- 深色品牌区块（`#1c1e54`）通过背景色对比创造沉浸式深度
- Ruby 到 magenta 过渡的渐变覆盖用于主区域装饰
- 阴影颜色 `rgba(0,55,112,0.08)`（`--hds-color-shadow-sm-top`）用于粘性元素的顶部边缘阴影

## 7. 做与不做

### 做
- 在每个文本元素上使用启用 `"ss01"` 的 sohne-var——样式集就是品牌本身
- 所有标题和正文使用 300 字重——轻盈感是标志性特征
- 对所有提升元素应用蓝调阴影（`rgba(50,50,93,0.25)`）
- 使用 `#061b31`（Deep Navy）作为标题颜色而非 `#000000`——温暖感很重要
- 将 border-radius 保持在 4px-8px 之间——保守的圆角是有意为之
- 任何等宽/金融数字显示使用 `"tnum"`
- 分层阴影：蓝调远景 + 中性近景形成深度视差
- 使用 `#533afd` 紫色作为主要交互/CTA 颜色

### 不做
- 不要为 sohne-var 标题使用 600-700 字重——300 字重是品牌声音
- 不要在卡片或按钮上使用大圆角（12px+，药丸形状）——Stripe 是保守的
- 不要使用中性灰色阴影——始终用蓝色调色（`rgba(50,50,93,...)`）
- 不要跳过任何 sohne-var 文本的 `"ss01"`——替代字形定义了个性
- 不要使用纯黑（`#000000`）作为标题——始终使用 `#061b31` Deep Navy
- 不要为交互元素使用暖调强调色（橙色、黄色）——紫色是主色
- 不要在展示尺寸使用正字间距——Stripe 的字距是紧凑的
- 不要将 magenta/ruby 强调色用于按钮或链接——它们仅用于装饰/渐变

1. 始终在 sohne-var 文本上启用 `font-feature-settings: "ss01"`——这是品牌的排版 DNA
