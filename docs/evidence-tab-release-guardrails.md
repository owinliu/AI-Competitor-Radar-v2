# APP版本更新 Tab 发布守则（仅更新数据，不改结构）

## 目标
保证该 Tab 在后续迭代中“内容持续更新、展示结构稳定”。

## 允许改动
1. 数据更新：
   - `data/app-version-timeline.json`
   - 与截图原话相关的数据文件
2. 文案更新：
   - 产品卡片中的原话卖点（仅替换内容）
   - 重点结论（保持3条短句结构）

## 禁止改动
1. 不新增流程说明文案（例如：抓取方式、命令、演示说明）。
2. 不引入内部协作术语（对齐/框架/模块/策略/方法等）。
3. 不把折叠明细改为默认展开。
4. 不将“各产品主打信息”改回多行松散布局（大屏应保持一行5卡片）。
5. 不在页面中展示技术细节（OCR、脚本、数据源调试信息）。

## 发布步骤（标准）
1. 更新数据与原话卖点。
2. 本地构建验证：`npm run build`。
3. 使用《evidence-tab-acceptance-checklist.md》逐项验收。
4. 提交并发布。

## 回滚原则
若发布后出现以下任一问题，回滚到上一版本：
- 页面出现过程说明或内部术语；
- 结构被改动（标题区、卡片区、折叠区布局异常）；
- 原话卖点与截图不一致且无法快速修正。
