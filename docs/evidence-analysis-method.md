# APP版本更新页（/evidence）分析方法固化 v1

更新时间：2026-04-29 20:14 (Asia/Shanghai)
适用页面：`/evidence/`（APP版本更新）
目标：把“证据采集 → 解析 → 对比 → 结论 → 页面展示”固定成可复用方法。

---

## 1. 数据源与链接获取（必须）

### 1.1 应用市场原始数据文件
- 文件：`data/app-version-timeline.json`
- 每条记录至少包含：
  - `competitor`
  - `version`
  - `currentVersionReleaseDate`
  - `releaseNotes`
  - `iconUrl`
  - `screenshotUrls[]`
  - `capturedAt`

### 1.2 本页实际使用的图片链接来源
- 结论证据图一律来自 `screenshotUrls[]`
- 在页面中展示的“上期/本期截图”必须可追溯为：
  - 上期：同竞品上一版本的 `screenshotUrls[]`
  - 本期：同竞品最新版本的 `screenshotUrls[]`

### 1.3 版本选择规则（固定）
- 按竞品分组后，先按 `version` 去重（保留 `capturedAt` 最新的一条）
- 再按 `currentVersionReleaseDate || capturedAt` 倒序
- 取：
  - `latest` = 最新版本
  - `previous` = 上一版本

---

## 2. 图片获取与本地可用性校验（必须）

### 2.1 展示优先级
1. `screenshotUrls[]` 可直接访问 → 直接展示
2. 若线上不可访问/丢失 → 落地到 `public/evidence/`，并改用本地路径

### 2.2 校验机制
- 对每个将展示的 URL 做可用性检查（HTTP 200 或本地文件存在）
- 任何失效 URL 不得静默展示；需：
  - 自动修复（重拷贝/替换）或
  - 明确标记“缺图待补”

### 2.3 命名规范（避免错配）
- 建议命名：`{竞品}-app-{期次}-{页面语义}.jpg`
- 禁止混用（如 首页 与 首页2）
- 涉及“首页/首页2”必须一一对位匹配，不得交叉

---

## 3. 文案提取与主打信息生成（从最新截图提取，禁止手写固定）

### 3.1 提取对象
- 每个竞品 `latest.screenshotUrls[]` 的前 N 张（默认 N=全部）
- 重点提取：
  - 首屏宣传大标题
  - 利益点文案（额度、利率、免息、审批速度、活动权益）
  - 强行动词（立即借、去领取、开通等）

### 3.2 提取方法
- OCR（如 tesseract）+ 规则清洗
- 清洗规则：
  - 去系统状态栏噪音词
  - 去低信息词（首页、我的、返回、更多等）
  - 同义合并（如“最高额度20万”与“额度最高20万”）

### 3.3 主打信息输出规则
- 每个竞品输出 3~5 条
- 必须可回溯到截图文本
- 不允许使用硬编码常量替代最新截图提取结果

---

## 4. 上期/本期截图对比分析逻辑（核心）

### 4.1 对比输入
- `previous.screenshotUrls[]` vs `latest.screenshotUrls[]`
- 对比粒度：多图集合级 + 单图内容级

### 4.2 对比维度（固定）
1. **新增**：本期新出现的主题/活动/入口表达
2. **替换**：同位置文案由 A 改为 B
3. **下线**：上期存在、本期消失
4. **稳定**：核心卖点延续，变化不明显

### 4.3 结论生成规则（必须“写具体变化”）
- 优先写“具体变化内容”，示例：
  - 新增：`新增“XX活动”主标题与利益点“XX”`
  - 替换：`主标题由“XX”替换为“XX”`
  - 下线：`上期“XX入口/活动”本期未出现`
- 若无明显变化：
  - 写“整体稳定”
  - 补充“提取到的关键文案”
- 禁止：仅写“高/中/低变化”但不说明内容

---

## 5. 顶部总结生成方法（本轮关键结论 + 差异总结）

### 5.1 输入
- 各产品主打信息（来自最新截图提取）
- 更新日志新增要点
- 上期/本期截图对比结论

### 5.2 输出规则
- 只总结与上述证据直接相关的内容
- 不写与页面证据无关的泛化结论
- 结构：
  - 本轮关键结论：3条（全局共性）
  - 差异总结：5条（逐产品一句）

---

## 6. 页面结构（固定版）

1. 标题区（时间范围、样本数、数据说明）
2. 本轮关键结论（证据驱动）
3. 差异总结（逐产品）
4. 各产品主打信息（来自最新截图提取）
5. 更新日志变化观察（辅助）
6. 证据明细表（主区）：
   - 列：`竞品 | 结论 | 上期截图(全量) | 本期截图(全量)`
   - 截图：可换行，支持点击查看

---

## 7. 质量闸门（发布前必须通过）

- [ ] 5个竞品是否齐全
- [ ] 每个竞品是否都有上期/本期截图列表（或明确缺图）
- [ ] 主打信息是否来自最新截图提取（非手写）
- [ ] 结论是否包含“具体变化内容”
- [ ] 顶部总结是否完全由本页证据支撑
- [ ] 关键图片链接是否可访问（避免线上丢图）

---

## 8. 失败与回滚策略

- 若 OCR 提取失败：
  - 结论降级为“稳定+可见文案片段”
  - 标记“建议人工复核”但不输出虚假变化
- 若图片缺失：
  - 先修复图片源，再发布结论
- 若版本错配（如 首页/首页2混配）：
  - 立即阻断发布，修正后再生成

---

## 9. 实施约束（长期）

- `/evidence/` 页分析结论一律遵守本方法
- 禁止再次回到“手写固定主打信息 + 非截图来源结论”
- 新增规则必须同步更新本文件并标注版本号
